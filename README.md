# Azov Breeze — production-ready starter for a small seaside stay business

Полноценный self-hosted проект на **Next.js + PostgreSQL + Prisma + Auth.js + Docker + Cloudflare Tunnel**.

Внутри уже есть:
- маркетинговая главная страница;
- каталог из 7 номеров с карточками;
- форма бронирования с занятостью по датам;
- предоплата через **ЮKassa**;
- личный кабинет гостя;
- админка с бронями, блокировками, услугами, лентой и базовой аналитикой;
- юридические страницы-заготовки;
- docker-compose для запуска на Ubuntu 24 + Cloudflare Tunnel по токену из `.env`.

## Стек
- Next.js 16 (App Router)
- React 19
- PostgreSQL 16
- Prisma ORM
- Auth.js (email/password + готовность к VK / Yandex OAuth)
- ЮKassa API
- Docker / Docker Compose

## Быстрый старт локально

```bash
cp .env.example .env
npm install
npm run db:migrate -- --name init
npm run db:seed
npm run dev
```

Открой `http://localhost:3000`.

## Демо-админ после seed
- Email: берётся из `ADMIN_EMAIL` в `.env` (по умолчанию `owner@example.com`)
- Пароль: берётся из `ADMIN_PASSWORD` в `.env` (по умолчанию `ChangeMe123!`)

Сразу после первого входа пароль нужно заменить на свой безопасный.

## Прод-развёртывание на Ubuntu 24

1. Скопируй проект на сервер.
2. Создай `.env` на основе `.env.example`.
3. Обязательно заполни:
   - `APP_URL`
   - `NEXTAUTH_SECRET`
   - `DATABASE_URL`
   - `CLOUDFLARE_TUNNEL_TOKEN`
   - `YOOKASSA_SHOP_ID`
   - `YOOKASSA_SECRET_KEY`
4. Запусти:

```bash
docker compose up -d --build
```

Cloudflare Tunnel контейнер уже запускается командой вида:

```bash
cloudflare/cloudflared:latest tunnel --no-autoupdate run --token ${CLOUDFLARE_TUNNEL_TOKEN}
```

## Что нужно заполнить перед реальным запуском
- реальные тексты и фото номеров;
- реквизиты владельца в юридических страницах;
- телефон, адрес, ссылки на мессенджеры;
- ключи ЮKassa;
- OAuth-ключи VK / Yandex, если нужны соцвходы;
- финальные правила возврата и предоплаты.

## Юридический чек-лист перед запуском
Этот проект даёт техническую базу, но перед запуском нужно финально проверить юридическую часть под свои реквизиты и схему работы:
- политика обработки персональных данных;
- пользовательское соглашение;
- условия бронирования / отмены / возврата;
- cookie policy и баннер согласия;
- уведомление об обработке персональных данных в Роскомнадзор при наличии обязанности;
- фактическое хранение баз персональных данных в РФ;
- корректная настройка чеков / 54-ФЗ в ЮKassa или через вашу кассу.

## Важные замечания по оплате
Если `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY` не заданы, система работает в demo-режиме: бронь подтверждается без реального эквайринга, чтобы можно было проверить пользовательский сценарий.
